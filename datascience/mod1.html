Unit 1: Introduction to AI – Evolution, Problems, Techniques, Applications

Mission Briefing: Understand the core identity of Artificial Intelligence, its origins, the challenges it tackles, the tools in its arsenal, and where it's deployed in the modern world.

What Is Artificial Intelligence?

AI is about building systems that think or act intelligently, performing tasks that typically require human intelligence. Think learning, problem-solving, perception, language, and decision-making. The goal is often to create machines that can adapt and learn like we do.

"Artificial Intelligence (AI) is the study of how to make computers do things at which, at the moment, people are better." - Rich (1983)

Evolution of AI

AI isn't new tech; it's been evolving for decades:

  • 1950s-60s (Genesis): Birth at Dartmouth Workshop (1956). Early programs like Logic Theorist tried symbolic reasoning. First neural network ideas (perceptrons).
  • 1970s-80s (Knowledge is Power): Rise of Expert Systems (e.g., MYCIN). Focus on encoding human expert knowledge into rules.
  • 1980s-90s (Learning Machines): Neural networks make a comeback (backpropagation). Machine learning algorithms (like decision trees) gain traction.
  • 2000s-Present (Data Deluge & Deep Dive): Big Data + powerful computers = Deep Learning explosion. Major breakthroughs in vision, NLP, games (AlphaGo). Statistical methods dominate.

It hasn't been a smooth ride: periods of hype ("AI Springs") were often followed by skepticism and funding cuts ("AI Winters").

AI Problems

AI tackles diverse challenges:

  1. Problem Solving & Search: Finding paths to goals (games, navigation).
  2. Knowledge Representation & Reasoning: Storing and using world knowledge.
  3. Learning: Improving performance from experience (Machine Learning).
  4. Perception: Interpreting sensors (vision, speech).
  5. Natural Language Processing (NLP): Understanding/generating human language.
  6. Planning: Creating action sequences for complex goals.
  7. Robotics: Building intelligent physical agents.

AI Techniques

Common methods used to solve AI problems:

  • Search Algorithms: Exploring possibilities (BFS, DFS, A*).
  • Logic-Based Methods: Formal reasoning (propositional/predicate logic).
  • Probabilistic Reasoning: Handling uncertainty (Bayesian networks).
  • Machine Learning: Learning from data (Neural Nets, Decision Trees).
  • Optimization: Finding the best solutions (Gradient Descent).

AI Applications

AI is everywhere:

  • Healthcare: Diagnosis, drug discovery.
  • Finance: Fraud detection, trading bots.
  • Transport: Self-driving cars, route planning.
  • Entertainment: Game AI (like in Valorant!), recommendation engines.
  • E-commerce: Personalization, chatbots.
  • ...and many more!

Unit 2: Various AI Domains (Introduction)

Mission Briefing: Identify the key specializations within the broad field of AI. Think of these as different agent classes, each with unique skills.

AI isn't one single thing; it's a collection of specialized fields:

  • Machine Learning (ML): The 'Learner'. Systems that improve from data/experience without explicit programming. (Core to modern AI).
  • Natural Language Processing (NLP): The 'Communicator'. Enables computers to understand and generate human language (text/speech). Powers chatbots, translation.
  • Computer Vision: The 'Observer'. Allows machines to "see" and interpret images/videos. Used in facial recognition, object detection.
  • Robotics: The 'Agent in the World'. Integrates AI (perception, planning) with physical hardware to act in the real world.
  • Expert Systems: The 'Specialist'. Emulates human expert decision-making in narrow domains using rules and knowledge bases. (Early AI success).
  • Planning and Scheduling: The 'Strategist'. Devises action sequences (plans) or allocates resources over time to achieve goals under constraints.
  • Knowledge Representation & Reasoning (KR&R): The 'Thinker'. Focuses on how to store knowledge formally and use it to derive new information.

Unit 3: Problem Solving Techniques - Search Algorithms, Knowledge Representation & Reasoning

Mission Briefing: Explore fundamental AI strategies: systematically searching for solutions and representing knowledge for reasoning.

Problem Solving as Search

Many AI tasks are like navigating a maze. We define:

  • State Space: All possible configurations (e.g., all possible chess board layouts).
  • Initial State: Where you start.
  • Goal State(s): Where you want to end up.
  • Operators/Actions: Moves that change the state (legal chess moves).
  • Solution: A sequence of actions (path) from start to goal.

AI uses search algorithms to explore this state space efficiently.

Common Search Strategies (Concepts)

  • Uninformed Search (Blind Search): Explores systematically without hints.
    • Breadth-First Search (BFS): Explores level by level. Finds the shortest path (in steps) but needs lots of memory. Like checking all adjacent rooms before going deeper.
    • Depth-First Search (DFS): Goes deep down one path first, then backtracks. Memory efficient, but might not find the shortest path and can get lost in long paths. Like following one corridor to the end.
  • Informed Search (Heuristic Search): Uses 'hints' (heuristics) to guess which paths are better.
    • Best-First Search: Expands the node that *looks* closest to the goal based on a heuristic function.
    • A* Search: Smartly combines the cost already traveled (g) with the estimated cost remaining (h). Often finds the shortest path efficiently if the heuristic is good (doesn't overestimate). Formula: f = g + h.

Knowledge Representation and Reasoning (KR&R)

How do AI systems 'know' things and 'think'?

  • Knowledge Representation (KR): Formalizing knowledge. Common ways:
    • Logic (Propositional/Predicate): Using formal rules (IF P THEN Q).
    • Semantic Networks: Nodes (concepts) linked by relations (IS-A, HAS-A). Like a mind map.
    • Frames: Templates for typical situations (e.g., a 'restaurant' frame with slots for 'cuisine', 'location').
    • Production Rules: IF-THEN rules often used in expert systems.
  • Reasoning: Using represented knowledge to deduce new facts or make decisions.

Unit 4: Problem Solving Techniques - Constraint Satisfaction Problems, Game Playing

Mission Briefing: Learn techniques for problems with strict rules (constraints) and for adversarial situations (games).

Constraint Satisfaction Problems (CSPs)

These problems involve finding a solution that satisfies a set of conditions or rules (constraints).

  • Components:
    • Variables: Things you need to decide values for.
    • Domains: Possible values for each variable.
    • Constraints: Rules limiting which values variables can take together.
  • Examples: Map coloring (neighboring regions different colors), Sudoku, scheduling tasks.
  • Solving (Concepts): Usually involves smart search (like backtracking) combined with Constraint Propagation (using constraints to eliminate impossible values early on).

Game Playing

AI in games (like Chess, Go, or even tactical shooters) often involves searching through possible future moves.

  • Game Tree: Represents all possible move sequences.
  • Minimax Algorithm: Assumes your opponent plays optimally to minimize your score. You choose the move that maximizes your minimum possible score. (Think worst-case scenario planning).
  • Alpha-Beta Pruning: A crucial optimization for Minimax. It smartly cuts off branches of the game tree that won't affect the final decision, saving huge amounts of computation.
  • Evaluation Function: When the game tree is too big (like Chess), AI uses a function to estimate how good a particular board state is, without searching all the way to the end.

Unit 5: Problem Solving Techniques - Machine Learning, Simulated Annealing

Mission Briefing: Introduce advanced techniques where systems learn solutions or use probabilistic methods to find good solutions in complex spaces.

Machine Learning in Problem Solving

Instead of explicitly programming a solution, Machine Learning (ML) lets the system learn it from data or experience.

  • ML can be used to:
    • Learn evaluation functions for games (how good is this board state?).
    • Learn which search strategies work best in certain situations (search control).
    • Learn optimal policies (sequences of actions) through trial and error (Reinforcement Learning - covered later).
  • ML excels when the problem is too complex to define exact rules or when the environment changes.

Simulated Annealing (Concept)

A powerful optimization technique for finding a very good (often globally optimal) solution in a huge, complex search space, especially one with many "local optima" (sub-optimal peaks).

  • Analogy: Like slowly cooling molten metal (annealing). High "temperature" allows random jumps, even to worse states, to escape local traps. As "temperature" drops, it settles towards a minimum energy (optimal) state.
  • How it Works: Starts randomly. Iteratively considers a neighbor state. Always accepts better states. Sometimes accepts *worse* states with a probability that decreases over time (based on "temperature").
  • Use in AI: Useful for complex optimization tasks in planning, scheduling, or even tuning ML models where standard methods get stuck.
